<h2>6. 느낀점</h2>
<h3>6-1. 어려웠던 부분</h3>
📌  Ajax의 비동기적인 특성을 이해하는 데 어려움을 겪었습니다.<br>
✔ 이전에는 순차적인 웹 개발 방식에 익숙했기 때문에, 동시에 여러 요청과 응답을 처리하는 개념을 처음 접할 때 혼란스러웠다.
Ajax를 통해 받은 데이터를 동적으로 처리하고 화면에 표시하는 작업이 새로웠고 JavaScript를 활용하여 데이터를 가공하고 표시하는 방법을 익히는 데 시간이 걸렸습니다.
하지만 Ajax를 다루는 과정에서 이러한 어려움을 경험하면서도 학습과 연습을 통해 점차 익숙해지고 개선해 나갈 수 있었고 지속적인 학습과 실전 경험을 통해 Ajax를 더욱 자신있게 다룰 수 있도록 노력해야겠다.

<h3>5-4. 총평</h3>
<h4>🌟 협업: 소통은 해도해도 모자르다 </h4>
팀원들과의 소통은 프로젝트 진행에 있어 가장 중요한 요소라고 생각하여 매일 팀원들과 작업 진행 상황을 공유하기 위해 노력했다. 하지만 깃허브에서 충돌이 발생하면 해결하는 데 상당한 시간이 소요된다는 것을 깨달았고, 작은 것 하나하나라도 상호 간에 소통해야 한다는 것을 배웠다. 따라서 향후 프로젝트에서는 소소한 사항이라도 상호 간에 소통하여 시간을 낭비하지 않고 더욱 발전된 서비스를 만들 것이다

<h4>🌟 미래: 다음은 어떤걸 해볼까? </h4>
이번 프로젝트에서는 다루고싶던 여러 api를 접해 사용해보게 되었다. 소셜로그인이라던지, 캘린더api, 카카오 주소api같은 것들이다.
개발자의 꽃은 다양한 api를 다루는 것이라 하던데, 추후 프로젝트에선 이를 기반으로 더욱더 다양한 라이브러리들과 api를 사용해보아야겠다.
